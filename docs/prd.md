2. 웹 기획서 형식 (ProoF 서비스 기획안)

이제는 같은 내용을 “웹/앱 서비스 기획서” 형식으로 정리해 드릴게요.
(FE/BE 개발자, 디자이너에게 전달하는 문서 느낌)

2-1. 서비스 개요

서비스명: ProoF (Proof of Process)

슬로건: “경험을 증명으로, 과정을 역량으로”

서비스 한 줄 정의

상경계열 대학생의 짧은 경험 로그를 AI로 분석해, 검증된 역량 키워드와 과정 중심 포트폴리오를 생성하는 커리어 플랫폼.

핵심 가치 제안 (Value Proposition)

학생에게: “회고를 못 쓰는 사람도, 말만 하면 정리되는 AI 회고/포트폴리오”

HR에게: “결과가 아니라 과정과 증거까지 함께 보는 신규 평가 데이터”

2-2. 타깃 정의 & 페르소나

Primary Target (초기 타깃)

상위권 대학 전략/마케팅/재무 학회 1~3학년

공모전·학회·프로젝트 경험이 많고, 취준을 대비해 정리된 포트폴리오가 필요하지만 기록 습관은 약한 사용자

대표 페르소나 예시

이름: 지민 (22세, 경영학과 3학년, 마케팅 학회 활동 중)

상황: 공모전·마케팅 프로젝트를 많이 했지만, 자소서 쓰려니 구체적인 스토리가 기억 안 남. 노션에 몇 번 정리하려다 포기.

니즈: “그냥 내가 한 말·톡을 기반으로 알아서 정리해 주고, 나중에 자소서 쓸 때 바로 쓸 수 있는 문장과 키워드가 있었으면 좋겠다.”

2-3. 핵심 기능 정의
1) 경험 로그 입력 & AI Ghostwriter

기능 설명

사용자가 하루 활동을 짧게 기록하면(AI 챗 또는 텍스트 입력), AI가 회고 초안과 키워드 후보를 자동 생성한다.

주요 요소

입력 타입: 텍스트, 해시태그, 날짜·프로젝트 태그

예시 입력:

“오늘 학회 회의에서 어제 만든 데이터 분석안 다 갈아엎음… #힘듦 #전략기획 #분석”

출력:

“초기 데이터 분석 결과 X 가설이 틀렸음을 발견하고, Y 방향으로 피봇을 제안함. 그 과정에서 팀원 설득과 자료 재정리가 필요했음.” + 키워드(#기획력, #문제정의, #데이터분석)

UI 포인트

채팅형 인터페이스: “오늘 뭐 했는지만 편하게 얘기해 주세요”

AI가 “어제 로그와 연결해서 이런 식으로 회고를 만들어 봤어요” 제안 → 사용자는 수정/저장만.

2) 프로젝트 & 타임라인 뷰

기능 설명

여러 날에 걸쳐 기록된 로그를 “프로젝트 단위”로 묶어 타임라인으로 보여준다.

주요 요소

프로젝트 생성/자동 그룹핑(동일 공모전 명, 동일 해시태그 기반)

타임라인: 시작일~종료일, 주요 사건(Log) 표시

프로젝트별 “요약 회고” 자동 생성 버튼

3) 3-Level KVS (Keyword Verification System)

Lv.1 Self-Claimed

사용자가 자신 로그에 역량 키워드 태그 직접 추가 또는 AI 추천 키워드 선택

UI: “이 경험에서 어떤 역량이 드러났나요?” + 추천 키워드 버튼

Lv.2 Peer-Endorsed

동일 프로젝트에 참여한 팀원이 확인할 수 있는 링크/알림 제공

UI 예시:

“OOO님과 [2025 00공모전] 활동이 함께 기록되어 있습니다. 서로의 역할을 확인하시겠어요?”

선택 옵션: “기획 리드”, “자료 조사”, “발표자” 등 역할 태그

Lv.3 Verified

사용자가 수상증명서, 인턴증명서, 발표자료 등 파일 업로드

백엔드: OCR → 텍스트 분석 → “수상 팀장”, “최우수상” 등 패턴 인식

UI:

“업로드하신 [2025_00공모전_수상증명서.pdf]에서 ‘최우수상’이 확인되었습니다. #기획력 키워드를 Lv.3로 업그레이드할까요?”

4) Process-Proof Portfolio 생성

기능 설명

프로젝트/직무별로, HR에게 제출 가능한 과정 중심 포트폴리오 PDF/웹 페이지를 생성한다.

포트폴리오 구성

상단: 지원자 기본 정보 + 역량 요약(#기획력 Lv.3, #데이터분석 Lv.2…)

본문:

핵심 경험 3~5개 카드 (문제 상황 → 시도/역할 → 결과/배운점)

각 카드 옆에 Lv.2/Lv.3 증거 리스트(팀원 인증, 파일 링크)

사용자 흐름

“전략기획 직무용 포트폴리오 만들기” 버튼 → 포함할 프로젝트 선택 → 템플릿 선택 → AI 자동 구성 → 사용자가 마지막 문장/디자인 수정 후 PDF 저장.

5) HR용 리포트 & 검색(후행 기능)

기능 설명 (Phase2~3)

기업 HR 계정에서 ProoF 인재풀을 검색하고, 지원자별 과정 리포트를 열람할 수 있는 웹 대시보드.

핵심 요소

검색 필터: 역량 키워드, 검증 레벨(Lv.2 이상), 활동 기간, 프로젝트 유형 등

지원자 상세 보기: 요약, 핵심 경험 카드, Lv.2/3 증거 목록, 파일 뷰어 링크

즐겨찾기·비공개 메모 기능

2-4. 정보 구조(IA)

상단 네비게이션 (학생용)

홈(Home): 오늘의 로그 입력, AI 추천 회고 보기

프로젝트(Project): 프로젝트 리스트, 타임라인, 프로젝트별 요약

역량(Strengths): 키워드 보드(키워드·레벨·관련 로그), 성장 추적 그래프

포트폴리오(Portfolio): 포트폴리오 생성·관리, PDF 다운로드

설정(My): 계정 설정, 학교/학회 정보, 알림 설정

HR용(별도 도메인 또는 서브 메뉴)

인재 검색

지원자 상세 리포트

채용 공고 관리

회사 계정 설정

2-5. 화면별 주요 요건(요약)

홈 화면

오늘 로그 입력 창(채팅형)

“어제/이번 주 주요 로그 요약” 카드

AI가 제안하는 “회고 업데이트 제안” 알림

프로젝트 상세 화면

타임라인(날짜별 로그 카드)

프로젝트 요약 텍스트(AI 자동 생성, 수동 수정 가능)

Lv.1~3 키워드 리스트 및 증거 파일

역량 보드 화면

키워드별 카드(#기획력, #데이터분석…)

각 카드에 레벨 표시(Lv.1~3), 관련 로그 개수, 최근 활동 날짜

클릭 시 관련 로그/프로젝트 리스트 팝업

포트폴리오 생성 화면

직무 선택(전략기획/마케팅/…)

포함할 프로젝트 선택(체크박스)

AI가 구성한 목차/본문 미리보기 + 수정 가능 에디터

PDF 다운로드/링크 공유 버튼

2-6. 데이터 구조(개략)

User

id, 이름, 학교, 학과, 학회, 학번, 목표 직무 등

Log

id, user_id, text, tags(해시태그), created_at, project_id

Project

id, title, 설명, 기간, 관련 로그 수 등

Keyword

id, 이름(#기획력 등), 도메인(전략/마케팅/재무)

UserKeyword

user_id, keyword_id, level(Lv.1~3), related_logs[], related_projects[]

Evidence

id, type(pdf, img 등), file_url, parsed_text, linked_keyword_id, linked_project_id

PeerEndorsement

from_user_id, to_user_id, project_id, role_tag, created_at

2-7. 단계별 개발 로드맵 (웹 기준)

MVP (0~3개월)

회원가입/로그인, 로그 입력, AI 회고 생성, 프로젝트 기본 그룹핑, 키워드 Lv.1 관리

v1 (3~6개월)

키워드 Lv.2 동료 인증 플로우, 역량 보드 화면, 프로젝트 타임라인 UI

v2 (6~12개월)

PDF 업로드 + OCR → Lv.3 자동 업그레이드, 포트폴리오 생성 기능, 기본 HR 리포트 뷰

v3 (1년+)

HR 검색 대시보드, 채용 공고 연동, 활동 추천/매칭 기능
