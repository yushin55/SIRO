[취준로그] 웹 애플리케이션 요구사항 명세서

1. 개요 (Overview)

프로젝트명: 취준로그 (Job Prep Log) - Simulation Ver.

목적: 사용자가 가상의 팀 프로젝트 시뮬레이션(채팅형)을 통해 본인의 직무 성향을 자연스럽게 발견하고, 이를 바탕으로 최적의 직무를 추천받아 커리어 로드맵을 관리할 수 있도록 함.

핵심 가치: 딱딱한 설문조사가 아닌, 몰입감 있는 대화형 인터페이스를 통한 직무 성향 분석.

2. 사용자 흐름 (User Flow)

랜딩 페이지: 서비스 소개 및 시뮬레이션 시작.

시뮬레이션 (채팅): 3단계(개인-팀-돌발) 시나리오를 순차적으로 진행하며 반응 선택.

결과 분석: 선택지에 따라 누적된 점수를 바탕으로 직무 성향 도출 및 AI 분석.

결과 리포트: 레이더 차트 및 AI 피드백 제공.

대시보드: 추천 직무 확인 및 관련 활동/프로젝트 관리.

3. 상세 기능 요구사항 (Functional Requirements)

3.1. 랜딩 페이지 (Landing Page)

헤더:

서비스 로고 (취준로그) 및 서브 텍스트 (SIMULATION) 표시.

네비게이션 메뉴 (시뮬레이션, 결과 리포트, 커리어 로드맵).

'MY' 프로필 아이콘 표시.

메인 섹션:

서비스 슬로건 및 설명 문구 표시.

[프로젝트 참여하기] 버튼: 클릭 시 시뮬레이션 페이지로 이동하며 화면 전환 애니메이션 적용.

3.2. 팀 프로젝트 시뮬레이션 (Simulation)

UI 레이아웃:

모바일 메신저(카카오톡 등) 스타일의 UI.

헤더: 프로젝트 방 이름("취준로그 런칭 프로젝트") 및 참여 인원 표시.

채팅 영역: 전체 화면 높이(h-full)를 활용하며, 하단 선택지 영역 확보를 위한 패딩(pb-48) 적용.

대화 로직:

NPC: PM(김팀장), 개발자(이대리), 마케터(박사원) 등 가상의 페르소나 등장.

시나리오 구성 (3-Phase Structure): 다음 3가지 상황을 순차적으로 진행하여 다각도로 성향을 분석함.

Phase 1: 개인 아이디어 발전 (Ideation): 혼자서 아이디어를 구상하고 논리를 발전시키는 과정 (창의성, 기획력, 분석력 중심).

Phase 2: 팀 프로젝트 협업 (Teamwork): 팀원들과 R&R을 정하고 소통하는 과정 (리더십, 팔로워십, 소통 방식 중심).

Phase 3: 돌발 상황 대처 (Crisis Management): 예상치 못한 문제(서버 다운, 일정 변경 등) 발생 시 해결 접근 방식 (문제 해결력, 순발력 중심).

자동 스크롤: 새로운 메시지나 선택지 등장 시 화면 최하단으로 부드럽게 스크롤 이동.

타이핑 효과: 봇 메시지 출력 전 '입력 중...' 애니메이션 노출 (1초 지연).

선택지 상호작용:

노출 방식: 화면 우측 하단에 사용자의 '답변 후보' 형태로 리스트업.

선택 효과: 사용자가 옵션을 클릭하면 해당 텍스트가 '나의 메시지'로 채팅창에 전송됨.

점수 산정: 각 선택지는 6가지 직무 역량 중 특정 분야에 가중치를 부여.

Job Buckets: MKT(마케팅), PM(기획), DATA(데이터), DEV(개발), DESIGN(디자인), PEOPLE(HR/소통).

3.3. 결과 분석 및 리포트 (Simulation Report)

직무 판별 로직:

시뮬레이션 완료 시 누적된 점수가 가장 높은 직무 코드를 도출.

각 코드별 직무명 및 설명 매핑 (예: MKT -> 그로스 마케터).

데이터 시각화:

Recharts 라이브러리의 Radar Chart(방사형 차트) 사용.

6각 역량 분포를 시각적으로 표현.

AI 피드백 (Gemini API):

Trigger: 결과 페이지 진입 시 자동 호출.

Prompt: 도출된 직무와 세부 점수 데이터를 기반으로 격려의 말과 추천 습관 생성.

Output: "AI COACH" 섹션에 텍스트로 표시.

3.4. 커리어 대시보드 (Project Dashboard)

상단 정보: 추천받은 직무(Best Match Job) 표시.

활동 추천 카드:

해당 직무와 관련된 추천 활동(사이드 프로젝트, 부트캠프 등) 카드 2종 표시.

내 프로젝트 관리:

현재 진행 중인 프로젝트가 없을 시 '빈 상태(Empty State)' 표시.

[새 프로젝트 추가] 버튼 제공 (기능 구현 확장성 확보).

  "text": "질문 내용...",

5. 비기능 요구사항 (Non-Functional Requirements)

5.1. UI/UX

반응형 디자인: 모바일 뷰를 기본으로 하되, 데스크톱 환경에서도 중앙 정렬된 모바일 형태 컨테이너 유지.

색상 테마:

Primary: Black (#000000) - 브랜드 아이덴티티.

Chat Background: Sky Blue Tone (#b2c7da).

User Bubble: Yellow (#ffeb33) - 카카오톡 스타일.

AI/Bot Bubble: White (#ffffff).

애니메이션:

animate-in, fade-in, slide-in-from-bottom 등을 활용한 자연스러운 요소 등장.

버튼 호버 및 클릭 시 스케일 변화 (active:scale-95).

5.2. 기술 스택 (Tech Stack)

Frontend: React.js

Styling: Tailwind CSS (유틸리티 클래스 활용)

Icons: Lucide React

Charts: Recharts (Radar Chart)

AI Model: Google Gemini 2.5 Flash (via REST API)

5.3. 성능 및 안정성

API 통신: Gemini API 호출 실패 시 "AI 분석 리포트를 작성중입니다..." 등의 로딩 상태 유지 혹은 에러 처리(기본 문구 출력).

스크롤 처리: 채팅 메시지가 쌓일 때 자동으로 최하단으로 이동(scrollIntoView)하여 사용자 경험 저해 방지.
